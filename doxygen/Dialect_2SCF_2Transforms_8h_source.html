69 /// Rewrite a for loop with bounds/step that potentially do not divide evenly
70 /// into a for loop where the step divides the iteration space evenly, followed
71 /// by another scf.for for the last (partial) iteration (if any; returned via
72 /// `partialIteration`). This transformation is called "loop peeling".
73 ///
74 /// This transformation is beneficial for a wide range of transformations such
75 /// as vectorization or loop tiling: It enables additional canonicalizations
76 /// inside the peeled loop body such as rewriting masked loads into unmaked
77 /// loads.
78 ///
79 /// E.g., assuming a lower bound of 0 (for illustration purposes):
80 /// ```
81 /// scf.for %iv = %c0 to %ub step %c4 {
82 /// (loop body)
83 /// }
84 /// ```
85 /// is rewritten into the following pseudo IR:
86 /// ```
87 /// %newUb = %ub - (%ub mod %c4)
88 /// scf.for %iv = %c0 to %newUb step %c4 {
89 /// (loop body)
90 /// }
91 /// scf.for %iv2 = %newUb to %ub {
92 /// (loop body)
93 /// }
94 /// ```
95 ///
96 /// After loop peeling, this function tries to simplify/canonicalize affine.min
97 /// and affine.max ops in the body of the peeled loop and in the body of the
98 /// partial iteration loop, taking advantage of the fact that the peeled loop
99 /// has only "full" iterations. This canonicalization is expected to enable
100 /// further canonicalization opportunities through other patterns.
101 ///
102 /// The return value indicates whether the loop was rewritten or not. Loops are
103 /// not rewritten if:
104 /// * Loop step size is 1 or
105 /// * Loop bounds and step size are static, and step already divides the
106 /// iteration space evenly.
107 ///
108 /// Note: This function rewrites the given scf.for loop in-place and creates a
109 /// new scf.for operation for the last iteration. It replaces all uses of the
110 /// unpeeled loop with the results of the newly generated scf.for.
111 LogicalResult peelAndCanonicalizeForLoop(RewriterBase &rewriter, ForOp forOp,
112  scf::ForOp &partialIteration);
113 
114 /// Try to simplify a min/max operation `op` after loop peeling. This function
115 /// can simplify min/max operations such as (ub is the previous upper bound of
116 /// the unpeeled loop):
117 /// ```
118 /// #map = affine_map<(d0)[s0, s1] -> (s0, -d0 + s1)>
119 /// %r = affine.min #affine.min #map(%iv)[%step, %ub]
120 /// ```
121 /// and rewrites them into (in the case the peeled loop):
122 /// ```
123 /// %r = %step
124 /// ```
125 /// min/max operations inside the partial iteration are rewritten in a similar
126 /// way.
127 LogicalResult rewritePeeledMinMaxOp(RewriterBase &rewriter, Operation *op,
128  AffineMap map, ValueRange operands,
129  bool isMin, Value iv, Value ub, Value step,
130  bool insideLoop);

174 /// Populate patterns for SCF software pipelining transformation.
175 /// This transformation generates the pipelined loop and doesn't do any
176 /// assumptions on the schedule dictated by the option structure.
177 /// Software pipelining is usually done in two part. The first part of
178 /// pipelining is to schedule the loop and assign a stage and cycle to each
179 /// operations. This is highly dependent on the target and is implemented as an
180 /// heuristic based on operation latencies, and other hardware characteristics.
181 /// The second part is to take the schedule and generate the pipelined loop as
182 /// well as the prologue and epilogue. It is independent of the target.
183 /// This pattern only implement the second part.
184 /// For example if we break a loop into 3 stages named S0, S1, S2 we would
185 /// generate the following code with the number in parenthesis the iteration
186 /// index:
187 /// S0(0) // Prologue
188 /// S0(1) S1(0) // Prologue
189 /// scf.for %I = %C0 to %N - 2 {
190 /// S0(I+2) S1(I+1) S2(I) // Pipelined kernel
191 /// }
192 /// S1(N) S2(N-1) // Epilogue
193 /// S2(N) // Epilogue
194 void populateSCFLoopPipeliningPatterns(RewritePatternSet &patterns,
195  const PipeliningOption &options);
