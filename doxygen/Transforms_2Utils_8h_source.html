35 /// Replaces all "dereferencing" uses of `oldMemRef` with `newMemRef` while
36 /// optionally remapping the old memref's indices using the supplied affine map,
37 /// `indexRemap`. The new memref could be of a different shape or rank.
38 /// `extraIndices` provides any additional access indices to be added to the
39 /// start.
40 ///
41 /// `indexRemap` remaps indices of the old memref access to a new set of indices
42 /// that are used to index the memref. Additional input operands to indexRemap
43 /// can be optionally provided in `extraOperands`, and they occupy the start
44 /// of its input list. `indexRemap`'s dimensional inputs are expected to
45 /// correspond to memref's indices, and its symbolic inputs if any should be
46 /// provided in `symbolOperands`.
47 ///
48 /// `domOpFilter`, if non-null, restricts the replacement to only those
49 /// operations that are dominated by the former; similarly, `postDomOpFilter`
50 /// restricts replacement to only those operations that are postdominated by it.
51 ///
52 /// 'allowNonDereferencingOps', if set, allows replacement of non-dereferencing
53 /// uses of a memref without any requirement for access index rewrites as long
54 /// as the user operation has the MemRefsNormalizable trait. The default value
55 /// of this flag is false.
56 ///
57 /// 'replaceInDeallocOp', if set, lets DeallocOp, a non-dereferencing user, to
58 /// also be a candidate for replacement. The default value of this flag is
59 /// false.
60 ///
61 /// Returns true on success and false if the replacement is not possible,
62 /// whenever a memref is used as an operand in a non-dereferencing context and
63 /// 'allowNonDereferencingOps' is false, except for dealloc's on the memref
64 /// which are left untouched. See comments at function definition for an
65 /// example.
66 //
67 // Ex: to replace load %A[%i, %j] with load %Abuf[%t mod 2, %ii - %i, %j]:
68 // The SSA value corresponding to '%t mod 2' should be in 'extraIndices', and
69 // index remap will perform (%i, %j) -> (%ii - %i, %j), i.e., indexRemap = (d0,
70 // d1, d2) -> (d0 - d1, d2), and %ii will be the extra operand. Without any
71 // extra operands, note that 'indexRemap' would just be applied to existing
72 // indices (%i, %j).
73 // TODO: allow extraIndices to be added at any position.
74 LogicalResult replaceAllMemRefUsesWith(
75  Value oldMemRef, Value newMemRef, ArrayRef<Value> extraIndices = {},
76  AffineMap indexRemap = AffineMap(), ArrayRef<Value> extraOperands = {},
77  ArrayRef<Value> symbolOperands = {}, Operation *domOpFilter = nullptr,
78  Operation *postDomOpFilter = nullptr, bool allowNonDereferencingOps = false,
79  bool replaceInDeallocOp = false);

118 /// Given an operation, inserts one or more single result affine apply
119 /// operations, results of which are exclusively used by this operation.
120 /// The operands of these newly created affine apply ops are
121 /// guaranteed to be loop iterators or terminal symbols of a function.
122 ///
123 /// Before
124 ///
125 /// affine.for %i = 0 to #map(%N)
126 /// %idx = affine.apply (d0) -> (d0 mod 2) (%i)
127 /// send %A[%idx], ...
128 /// %v = "compute"(%idx, ...)
129 ///
130 /// After
131 ///
132 /// affine.for %i = 0 to #map(%N)
133 /// %idx = affine.apply (d0) -> (d0 mod 2) (%i)
134 /// send %A[%idx], ...
135 /// %idx_ = affine.apply (d0) -> (d0 mod 2) (%i)
136 /// %v = "compute"(%idx_, ...)
137 
138 /// This allows the application of different transformations on send and
139 /// compute (for eg. different shifts/delays)
140 ///
141 /// Fills `sliceOps` with the list of affine.apply operations.
142 /// In the following cases, `sliceOps` remains empty:
143 /// 1. If none of opInst's operands were the result of an affine.apply
144 /// (i.e., there was no affine computation slice to create).
145 /// 2. If all the affine.apply op's supplying operands to this opInst did not
146 /// have any uses other than those in this opInst.
147 void createAffineComputationSlice(Operation *opInst,
148  SmallVectorImpl<AffineApplyOp> *sliceOps);
