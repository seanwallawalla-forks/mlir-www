34 /// Create a clone of `loop` with `newIterOperands` added as new initialization
35 /// values and `newYieldedValues` added as new yielded values. The returned
36 /// ForOp has `newYieldedValues.size()` new result values. The `loop` induction
37 /// variable and `newIterOperands` are remapped to the new induction variable
38 /// and the new entry block arguments respectively.
39 ///
40 /// Additionally, if `replaceLoopResults` is true, all uses of
41 /// `loop.getResults()` are replaced with the first `loop.getNumResults()`
42 /// return values respectively. This additional replacement is provided as a
43 /// convenience to update the consumers of `loop`, in the case e.g. when `loop`
44 /// is soon to be deleted.
45 ///
46 /// Return the cloned loop.
47 ///
48 /// This convenience function is useful to factorize common mechanisms related
49 /// to hoisting roundtrips to memory into yields. It does not perform any
50 /// legality checks.
51 ///
52 /// Prerequisite: `newYieldedValues.size() == newYieldedValues.size()`.
53 scf::ForOp cloneWithNewYields(OpBuilder &b, scf::ForOp loop,
54  ValueRange newIterOperands,
55  ValueRange newYieldedValues,
56  bool replaceLoopResults = true);
